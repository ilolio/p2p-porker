# ♠ P2P Poker ♥

ブラウザだけで遊べる **P2P テキサスホールデム** ポーカーゲームです。  
サーバー不要。WebRTC（PeerJS）を使ってプレイヤー同士が直接接続します。

**🎮 プレイはこちら → https://ilolio.github.io/p2p-porker/**

---

## 特徴

- **サーバーレス** — バックエンド不要。GitHub Pages の静的ファイルのみで動作
- **P2P 通信** — [PeerJS](https://peerjs.com/)（WebRTC）でプレイヤーが直接接続
- **ホスト権限モデル** — ゲームの進行はホストが管理。他のプレイヤーはホストに接続
- **テキサスホールデム** — プリフロップ / フロップ / ターン / リバー / ショーダウンの完全実装
- **手役判定** — ロイヤルフラッシュ〜ハイカードまでの全役に対応（7枚から最強の5枚を選択）

## 遊び方

1. **[ゲームを開く](https://ilolio.github.io/p2p-porker/)**
2. プレイヤー名を入力する
3. **ルームを作成（ホスト）** または **Room ID を入力して参加**
4. 全員が揃ったらホストが「ゲーム開始」を押す
5. テキサスホールデムを楽しむ！

> **Room ID の共有方法**  
> ホストがルームを作成すると Room ID が表示されます。  
> Room ID をクリックするとクリップボードにコピーされるので、他のプレイヤーに送ってください。

### 使用ライブラリ

- **[PeerJS 1.5.4](https://peerjs.com/)** — WebRTC の P2P 接続を抽象化
- STUN サーバー: `stun.l.google.com:19302`

## ローカルで動かす

```bash
# HTTPサーバーが必要（file:// では WebRTC が動作しないため）
npx serve .
# または
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080` を開く。

## 制限事項

- NAT 越えが困難な環境では接続できない場合があります（TURN サーバー未使用）
- サイドポットは未実装（オールインが複数いる場合は勝者総取り）
- 同一ネットワーク内での利用が最も安定します
